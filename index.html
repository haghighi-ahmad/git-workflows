Gitflow

Gitflow


  • Main branches
    • master
    • develop
  • Supporting branches
    • Feature branches
    • Release branches
    • Hotfix branches
Gitflow: "develop" branch
Gitflow: "develop" branch

  • Infinite lifetime
  • HEAD of master:
    • production-ready state
      •
  • HEAD of develop:
    • Latest delivered development changes for the next release
    • This is where any automatic nightly builds are built from.
      •
Gitflow: "feature" branches
Gitflow: "feature" branch

  • Limit lifetime
  • May branch off from:
    • develop
      •
  • Must merge back into:
    • develop
      •
  • Branch naming convention:
    • anything except master, develop, release-*, or hotfix-*
      •
Gitflow: "feature" branches - Merge with --no-ff
Gitflow: "release" branches
Gitflow: "release" branch

  • May branch off from:
    • develop
      •
  • Must merge back into:
    • develop and master
      •
  • Branch naming convention:
    • release-*
      •
Gitflow: "hotfix" branches
Gitflow: "hotfix" branch

  • May branch off from:
    • master
      •
  • Must merge back into:
    • develop and master
      •
  • Branch naming convention:
    • hotfix-*
      •

GitHub flow

GitHub flow

GitHub flow

  • Anything in the master branch is deployable
  • Create descriptive branches off of master
  • Push to named branches constantly
  • Open a pull request at any time
  • Merge only after pull request review
  • Deploy immediately after review
